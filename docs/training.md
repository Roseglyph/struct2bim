# Local model training

Model training is optional and is intentionally separate from the base development environment. The repository includes the dataset formats, configurations, checkpoint handling, resume behavior and evaluation entry points required for a user-run experiment.

No custom model weights or detector metrics are included with the repository.

## 1. Generate and validate a dataset

```powershell
struct2bim generate --config configs/curricula/reference.yaml
struct2bim validate-dataset --dataset outputs/dataset
```

Keep all generated datasets under `outputs/` or `datasets/`; both are ignored by Git.

## 2. Create a separate training environment

Create a second virtual environment rather than adding large machine-learning packages to the base environment:

```powershell
.\.tools\uv\uv.exe venv .venv-training --python 3.11
.\.tools\uv\uv.exe pip install --python .venv-training\Scripts\python.exe -r requirements-training.txt
.\.tools\uv\uv.exe pip install --python .venv-training\Scripts\python.exe -e .
```

Install the PyTorch build appropriate for the machine's GPU and driver when the standard package source is not suitable. The full CUDA Toolkit is normally unnecessary when using an official PyTorch binary distribution.

## 3. Start training

```powershell
.\.venv-training\Scripts\python.exe -m struct2bim.cli train `
  --config configs/training/columns-seg.yaml
```

Run directories, weights and checkpoints are ignored by Git.

## 4. Resume an interrupted run

Set `resume_checkpoint` in a copied local configuration:

```yaml
resume_checkpoint: runs/segment/columns-seg/weights/last.pt
```

Then run the same command. The runner loads that checkpoint and enables Ultralytics resume behavior.

## 5. Reporting results

Only metrics produced by an actual checkpoint and recorded dataset manifest should be published. Synthetic ground-truth overlays generated by the showcase are not detector predictions.

Evaluate a supplied checkpoint against the generated test split:

```powershell
struct2bim evaluate --weights path\to\best.pt `
  --dataset outputs\dataset `
  --data outputs\dataset\segment\dataset.yaml
```

Run inference on an image, a multi-page PDF, or a DXF drawing:

```powershell
struct2bim infer --source drawing.pdf --weights path\to\best.pt
```

Without a scale, predictions remain in pixel space. Supply a measured calibration only when it is known to create canonical geometry and IFC:

```powershell
struct2bim infer --source drawing.pdf --weights path\to\best.pt --mm-per-pixel 2.5
```

DWG is intentionally rejected with guidance to export DXF or PDF first. The repository does not claim reliable open-source DWG round-trip support.
